# Synthetic Data Creation

This directory contains the data generation pipeline used to create step-by-step image editing pairs for training the compositional image generation model.

The pipeline is:

1. Generate full compositional images with FLUX.1-dev.
2. Use Grounded-SAM2/GroundingDINO to detect objects and save masks/boxes.
3. Use VQAScore to reject or relabel images whose objects, relations, style, or background do not match.
4. Use Qwen-Image-Edit with a `Remove ...` instruction to remove selected objects from the full image.
5. Collate the resulting examples into `{id}_condition.png`, `{id}.png`, `{id}.json` training triples.

## Stages

| Stage | Script | Input | Output |
| --- | --- | --- | --- |
| 1. Generate and detect | `generate_flux_images_and_detect_objects.py` | Sampled objects, relations, style, and background | FLUX image, prompt metadata, Grounded-SAM2 masks, boxes, and detection labels |
| 2. Reject and relabel | `filter_and_relabel_with_vqa.py` | Stage 1 folders | `_relabelled.json`, relabelled masks/boxes, and rejected samples skipped from later stages |
| 3. Remove objects | `remove_objects_with_qwen_edit.py` | Stage 2 relabelled samples and a Qwen-Image-Edit server | Edited intermediate images plus `qwen_remove.json` with removed objects, removed relations, and the forward addition prompt |
| 4. Collate training pairs | `collate_step_by_step_training_pairs.py` | Stage 3 output folders | `{id}_condition.png`, `{id}.png`, and `{id}.json` training triples |

The generated training examples are forward step-by-step supervision pairs. Stage 3 removes objects from a complete image; Stage 4 flips those removal trajectories into addition examples, so the model learns to add the missing object or relation back into the scene.

## Setup

```bash
cd image_gen/train_step_by_step_model/synthetic_data_creation
bash setup.sh
micromamba activate iter-refine-data-gen
```

Grounded-SAM2 is required by Step 1:

```bash
git clone https://github.com/IDEA-Research/Grounded-SAM-2.git Grounded_SAM2
```

Download the SAM2 and GroundingDINO checkpoints expected by the scripts:

```text
Grounded_SAM2/checkpoints/sam2.1_hiera_large.pt
Grounded_SAM2/gdino_checkpoints/groundingdino_swint_ogc.pth
```

## Start Qwen-Image-Edit

If using the local vLLM-Omni environment from the image inference code:

```bash
bash scripts/serve_qwen_image_edit.sh 8092 Qwen/Qwen-Image-Edit
```

The helper expects `vllm-omni` on `PATH`. If it is installed in a separate environment, set one of:

```bash
export VLLM_OMNI_BIN=/path/to/vllm-omni
# or
export VLLM_OMNI_ENV_DIR=/path/to/micromamba/env
```

The data-generation script uses `QWEN_EDIT_BASE_URL`, defaulting to:

```bash
export QWEN_EDIT_BASE_URL=http://localhost:8092/v1
```

## Run The Pipeline

Small smoke run:

```bash
bash scripts/run_synthetic_data_generation_example.sh
```

Manual steps:

```bash
DATA_ROOT=outputs/synthetic_flux_qwen
SOURCE_DIR=$DATA_ROOT/dataset_512_sbatch0

python generate_flux_images_and_detect_objects.py \
  --cuda 0 \
  --dataset_folder_name "$SOURCE_DIR" \
  --max_num_rels 4 \
  --num_samples_to_generate_per_k 100

python filter_and_relabel_with_vqa.py \
  --input_dir "$SOURCE_DIR" \
  --verbose False \
  --skip_if_item_not_detected True \
  --use_spatial_relations_if_norelabel False

python remove_objects_with_qwen_edit.py \
  --source_folder "$SOURCE_DIR" \
  --qwen-edit-base-url "$QWEN_EDIT_BASE_URL" \
  --max_items_to_select 2

python collate_step_by_step_training_pairs.py \
  --input_primary_dir "$DATA_ROOT" \
  --output_dir "$DATA_ROOT/collated_train_pairs"
```

## Output Format

The final collated dataset contains:

```text
0_condition.png
0.png
0.json
1_condition.png
1.png
1.json
...
```

For the first step, `*_condition.png` is a black image and `*.png` is either the full target image or the Qwen-removed intermediate image. For the next step, `*_condition.png` is the Qwen-removed image and `*.png` is the original full image.

The JSON stores the local step prompt, full context prompt, original objects/relations, removed objects/relations, and whether the sample is the first step.
